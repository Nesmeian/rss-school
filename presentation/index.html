React

React

It is a declarative, efficient, and flexible JavaScript library for creating user interfaces. It allows us to assemble a complex UI from small isolated pieces of code called "components".

React was developed by Facebook in March 2013

React was created to solve the problem of efficiently updating the user interface in real-time without requiring the entire page to be reloaded.
The history of React development is quite extensive, so let's focus on it for a moment.

React 0.14(2015)

In this version, stateless components were added, making the code more declarative and testing easier.
A stateless component, also known as a functional component, is a fundamental concept in React that represents a UI element without any internal state management. In simpler terms, a stateless component is a function that takes props as input and returns JSX (JavaScript XML) to describe the UI.

React 0.3 (2013)

The first release of React introduced basic concepts such as the virtual DOM and unidirectional data flow. These principles ensured that the interface was effectively updated without the need for a complete page re-rendering.

React 16.0 (2017)

The emergence of Fiber, a new architecture of the React internal device.
Fiber's goal is to increase productivity when developing tasks such as animation, organizing elements on a page, and moving elements. Its main feature is incremental rendering: the ability to divide the rendering work into units and distribute them among multiple frames.

React 16.3 (2018)

Context has been added, and the lifecycle system has been improved.
Context provides a way to pass data through the component tree without having to pass props down manually at every level.

React 16.8(2019)

Hooks were added - functional components providing access to state and the possibility of using lifecycle methods.
Hooks allow using states and other features of React without using classes.

Advantages and Disadvantages

Advantages

  • Quick because it uses Virtual DOM
  • A huge number of open libraries
  • One-Way Binding
  • React Native
  • Advanced and most useful React Dev tools
  • Has a low entry threshold

VDOM

The virtual DOM (VDOM) is a programming concept where an ideal, or “virtual”, representation of a UI is kept in memory and synced with the “real” DOM by a library such as ReactDOM. This process is called reconciliation.
In simple terms, VDOM compares its nodes and nodes of the DOM; if something is different, then it redoes only these nodes and doesn't touch the entire tree. This is faster than manipulation with DOM.

A huge number of open libraries

A large number of libraries have been developed for React, which allow you to work without problems even with very complex animations, and not only with it.
Examples include Formik, React Popper, React animations, React Icons.

One-Way Binding

One-way binding means that the binding happens in one direction. In this case, changes in the data automatically update the UI, but changes in the UI do not automatically update the data. That’s why it is referred to as one-way data binding.

React Native

React Native is a popular JavaScript-based mobile application platform that allows creating native mobile applications for iOS and Android. Also, this framework allows creating applications for different platforms using the same code base.

React Dev Instruments

These tools help coding smoothly and easier.
Examples include React Datasheet, which allows using math formulas, React Native for Web, which adds progress bars, buttons, and a lot of useful things, Reactide, which is an IDE tool for working with React, among others.

has a low entry threshold

compared to other frameworks and libraries, React is relatively easy to use, but it takes a long time to master it.

Disadvantages

There are not many of them, but they are worth mentioning.
  • Doesn't work out of the box
  • There is no division into modules
